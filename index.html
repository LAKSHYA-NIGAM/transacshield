What it validates

  • Phone numbers against country-specific rules (India: 10, Singapore: 8)
  • Date/time values against calendar-safe formats
  • Payment modes against a configurable allowed list
  • Amounts as non-negative numeric values
  • Required fields (order ID) and duplicate detection
  • Email fields for standard syntactic formats

What it exports

  • Cleaned CSV — valid rows only, with normalized values
  • Validation Report — all rows with status/error tags
  • Chunked files — split cleaned data into custom sizes

Assumptions

  • Input files are UTF-8 encoded CSVs with a header row
  • Default phone country is India (IN) unless configured
  • All processing is client-side — no data leaves your browser
  • Duplicate checking uses mapped unique identity columns
