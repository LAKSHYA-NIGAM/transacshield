About This Validator

What it validates

  • Phone numbers against country-specific digit rules (India: 10, Singapore: 8)
  • Date/time values against predefined calendar-safe formats
  • Payment modes against a configurable allowed-values list
  • Amounts as non-negative numeric values
  • Required fields (order ID) and duplicate detection

What it exports

  • Cleaned CSV — valid rows only, with normalized values
  • Validation Report — all rows with error/warning annotations
  • Chunked files — cleaned data split into configurable-size parts

Assumptions

  • Input files are UTF-8 encoded CSVs with a header row
  • The default phone country is India (IN) unless changed in config
  • All processing is client-side — no data leaves the browser
  • Duplicate detection uses order_id (or order_id + product_id if both are mapped)
